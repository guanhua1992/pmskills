# 语义质量评审清单（Semantic Quality Rubric）

本清单用于人工评审与校验器建议，作为确定性检查的补充。

## 价值质量

强：

- 问题与方案分离；
- 用户价值与业务价值都可见；
- 成功指标与护栏指标可量化；
- 不做这件事有明确后果。

弱：

- 原样接受一个功能请求；
- 主要理由是"提升体验"；
- 价值未绑定到用户行为或业务结果。

## 需求质量

强：

- 每个 `FR-*` 描述一个可观察行为；
- 每个需求映射到一个场景与目标；
- 业务规则是显式的；
- 验收标准覆盖正常、备选、异常、权限与边界路径。

弱：

- 需求把多个行为揉在一起；
- UX 偏好与业务规则混在一起；
- 工程必须推断数据、权限或状态行为。

## Vibe Coding 质量

强：

- 系统边界清晰；
- 输入与输出完整；
- 模块有各自职责；
- 完成标准可测试；
- 列出了 AI 非决策项。

弱：

- prompt 只说"构建这个功能"；
- AI 必须自行臆造产品策略；
- 用外观而非行为来判断成功。

## 评审质量

强：

- 记录了评审类型、评审人、通过条件、发现与行动项；
- 阻断项是显式的；
- `approved` 状态与已解决的评审决策绑定。

弱：

- 评审只是一段聊天小结；
- 行动项无负责人；
- 批准是默示的，而非有记录的。
