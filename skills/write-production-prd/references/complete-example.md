# 完整示例：周期任务管控

这是一个精简示例，用于演示必需的稳定标记与需求格式。真实 PRD 应根据其证据与风险展开到相应深度。叙述用中文，标记 `[PRD-*]`、ID、状态值保持英文。

## 文档控制 [PRD-CONTROL]

- 状态：review-ready
- 深度：standard
- 产品画像：general
- 负责人：产品
- 受众：工程、设计、QA、运维

## 背景 [PRD-BACKGROUND]

`SRC-001` 显示运营用户每周手动重建周期性任务。已确认的问题是漏执行与重复劳动，而不仅仅是"缺少一个重复按钮"。

## 目标 [PRD-GOALS]

| ID | 目标 | 基线 | 目标值 | 护栏 |
|---|---|---:|---:|---|
| GOAL-001 | 减少手动重建 | 18 分钟/周/人 | <= 5 分钟/周/人 | 不增加误建任务 |

## 用户与 JTBD [PRD-USERS]

- SCN-001：当运营负责人需要创建可预期的每周固定事务时，希望系统自动排定后续实例，使该事务不被遗忘。

## 范围 [PRD-SCOPE]

- 范围内：每周重复、暂停/恢复、编辑单个实例、编辑后续实例。
- 范围外：日历同步。

## 流程 [PRD-FLOWS]

```mermaid
flowchart LR
    A[创建任务] --> B[启用重复]
    B --> C[选择每周计划]
    C --> D[确认]
    D --> E[创建系列与首个实例]
```

## 功能需求 [PRD-REQUIREMENTS]

### FR-001: 创建每周周期任务

- 优先级：P0
- 追踪至：GOAL-001, SCN-001, SRC-001
- 业务规则：BR-001
- 需求：当用户确认一个有效的每周计划后，系统必须创建一个周期系列及其首个任务实例。
- 验收标准：
  - AC-001: 假设存在有效任务与每周计划，当用户确认创建，则可见一个系列和一个首个实例。
  - AC-002: 假设每周计划不完整，当用户确认创建，则创建被阻止并标出缺失字段。

## 业务规则 [PRD-RULES]

- BR-001：每周计划必须至少选中一个工作日。

## 异常 [PRD-EXCEPTIONS]

- 后台重复执行不得创建重复实例。
- 若任务创建失败，界面保留用户输入并提供重试操作。

## 数据 [PRD-DATA]

- 存储系列标识、重复规则、时区、状态与下次生成时间。

## 权限 [PRD-PERMISSIONS]

- 能在某项目中创建任务的用户，即可在该项目中创建周期任务。

## 集成 [PRD-INTEGRATIONS]

- 不适用，因为已确认的版本不与外部日历集成。

## 非功能需求 [PRD-NFR]

### NFR-001: 生成幂等

- 类别：reliability
- 追踪至：GOAL-001, SRC-001
- 目标：对同一生成任务连续执行 2 次，产生的重复实例数为 0 个。
- 验证：在集成测试中将同一调度任务执行 2 次。

## 指标与监控 [PRD-METRICS]

- 追踪：系列已创建、实例已生成、生成失败、系列已暂停。
- 告警：15 分钟内生成失败率超过 1% 时触发。

## 发布 [PRD-RELEASE]

- 灰度顺序：内部用户 -> 10% -> 50% -> 100%。
- 平台与兼容性：现代浏览器（Chromium 内核）Web 后台。
- 回滚：禁用新系列创建，同时保留既有数据。

## 验收 [PRD-ACCEPTANCE]

- 所有 P0 需求通过正常、异常、权限测试。
- 运维确认：无需产品团队协助即可创建并暂停一个每周固定事务。

## 风险与决策 [PRD-RISKS]

- 风险：用户可能混淆"编辑单个实例"与"编辑后续实例"。
- 缓解：提供明确的选择对话框，并在大范围灰度前做可用性测试。

## AI 开发意图 [PRD-VIBE-SPEC]

| 项 | 答复 |
|---|---|
| 构建意图 | 构建"周期任务自动排定"的产品能力 |
| 用户 | 运营负责人；排除只读访客 |
| 模块 | 周期系列管理、实例生成调度 |
| 输入 | 任务、每周计划、用户操作 |
| 输出 | 周期系列、任务实例、生成失败事件 |
| 边界 | 不含日历同步；明确在系统之外 |
| 完成标准 | 用户能创建/暂停每周固定事务，且生成幂等 |
| AI 不得决策 | 重复规则的产品策略、数据保留期；禁止 AI 自行决定，须产品确认 |

| FR-001 | GOAL-001 | SCN-001 | SRC-001 | AC-001, AC-002 |
